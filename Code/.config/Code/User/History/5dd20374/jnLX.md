## Shell Configuration (Fish/Zsh)
When editing or creating fish or zsh configuration files:
- Ensure proper shell syntax and follow best practices for shell scripting.
- Consider cross-compatibility if configurations need to work in both fish and zsh environments.
- Use appropriate quoting and escaping for variables and commands.

## Hyprland Configuration
When working with Hyprland configuration files:
- Be mindful of Hyprland's configuration syntax, including window rules, keybindings, and animations.
- Consider performance implications when adding complex animations or rules.
- Ensure configurations align with Wayland standards.

## General Dotfiles Practices
- Follow XDG Base Directory Specification for configuration file locations.
- Use the omarchy skill for theming-related customizations in applications like alacritty, cava, and other themed components.
- Maintain consistency across related configurations for a cohesive desktop experience.
